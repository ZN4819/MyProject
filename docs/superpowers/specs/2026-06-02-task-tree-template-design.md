# 项目任务树模板设计

## 目标

将现有“工作流模板”从单纯状态列表扩展为两层模板体系：

- 状态工作流模板：保留现有待办、进行中、阻塞、完成等状态定义。
- 项目任务树模板：根据项目类型、标签或关键词推荐，用于一键生成项目任务树，并记录模板节点之间的前置依赖。

MVP 采用“自动推荐 + 手动确认”：创建或编辑项目时，系统根据项目类型、标签和名称推荐任务树模板；用户确认后，平台将模板节点复制成真实任务。

## 范围

本阶段实现：

- 项目新增 `projectType` 字段，用于模板匹配。
- 新增任务树模板、模板节点、模板依赖关系三类数据。
- 支持创建任务树模板，包含多级节点和前置依赖。
- 支持按项目信息推荐模板。
- 支持将模板套用到项目，生成真实任务树。
- 支持在界面查看模板、推荐模板，并从项目详情页确认套用。

本阶段不实现：

- 并行分组的复杂排程。
- 任务完成后自动创建后续任务。
- 可视化拖拽编辑模板关系图。
- 阻止用户手动推进有前置依赖未完成的任务。

## 数据模型

`Project` 增加：

- `projectType: string | null`

`TaskTreeTemplate`：

- `id`
- `name`
- `description`
- `projectType`
- `matchKeywords`
- `workflowTemplateId`
- `createdAt`
- `updatedAt`

`TaskTemplateNode`：

- `id`
- `templateId`
- `title`
- `description`
- `parentId`
- `workflowTemplateId`
- `defaultStatusKey`
- `priority`
- `sortOrder`
- `tags`

`TaskTemplateDependency`：

- `id`
- `templateId`
- `fromNodeId`
- `toNodeId`
- `type`

MVP 依赖类型只使用 `finish_to_start`，表示 `toNodeId` 对应任务应等待 `fromNodeId` 对应任务完成。

真实任务增加：

- `templateNodeId`

该字段记录任务由哪个模板节点生成，方便生成依赖映射和后续追踪。

## 核心流程

1. 用户创建或编辑项目，填写项目类型、标签、名称。
2. 系统读取任务树模板，根据项目类型、标签、名称进行匹配评分。
3. 项目详情展示推荐模板。
4. 用户点击“套用模板”。
5. 后端将模板节点按父子层级复制为任务。
6. 后端按模板依赖生成真实任务依赖关系。
7. 项目任务树刷新，显示已生成任务。

## 匹配规则

推荐规则按分值排序：

- 项目类型与模板项目类型完全匹配：+60
- 项目名称包含模板关键词：每个 +15
- 项目标签包含模板关键词：每个 +20
- 项目描述包含模板关键词：每个 +10

分数大于 0 的模板作为推荐结果。分数相同按模板名称排序。

## 错误处理

- 模板不存在时返回 404。
- 项目不存在时返回 404。
- 套用空模板时返回 400。
- 同一项目重复套用同一模板时允许生成新任务，但前端给出提示文案，避免误操作。
- 模板依赖若引用缺失节点，套用时跳过该条依赖。

## 测试

- 领域测试：模板推荐评分、模板节点构建任务树、依赖映射。
- Store 测试：创建模板、推荐模板、套用模板生成多级任务和依赖。
- UI 验证：项目详情页可填写项目类型、查看推荐模板、套用后任务树出现模板任务。
