# 人员管理与任务分配设计

## 目标

在现有个人项目管理平台中增加测评人员档案管理和任务多人分配能力，同时完善任务开始时间记录，并限制任务详情中的项目归属修改。

## 功能范围

### 测评人员管理

设置页面新增“测评人员管理”区域，支持：

- 新增测评人员，字段包含姓名、证书编号。
- 编辑未删除人员的姓名和证书编号。
- 删除人员采用软删除，不物理移除人员记录。
- 已删除人员保留历史任务关联，但不再出现在新任务的人员选择列表中。
- 设置页继续展示已删除人员，并明确标记为“已删除”；已删除人员不可继续编辑。

姓名和证书编号均为必填字段。证书编号在未删除及已删除人员中保持唯一，避免历史人员被重复录入后造成任务归属混淆。

### 任务人员分配

任务详情面板增加“分配人员”多选控件：

- 一个任务可以分配零名、一名或多名测评人员。
- 下拉候选项只包含未删除人员。
- 当前任务已经关联的已删除人员仍需展示。
- 已删除人员名称显示为 `姓名！`，并提供“该人员已删除”的辅助说明或悬停提示。
- 用户可以从任务上移除已删除人员，但不能重新添加已删除人员。
- 保存任务详情时，同步更新任务与人员的关联关系。

### 任务开始时间

任务增加可为空的开始时间字段：

- 任务首次进入工作流的“进行中”状态时，如果开始时间为空，自动记录当前时间。
- 后续离开并再次进入“进行中”状态时，不覆盖已有开始时间。
- 用户可以在任务详情面板手动修改或清空开始时间。
- 手动修改后的值不会被后续状态流转覆盖。

“进行中”状态以当前工作流状态键 `in_progress` 判断。

### 项目归属只读

项目任务详情中的归属项目只读展示：

- 不允许通过任务详情修改 `projectId`。
- 后端常规任务更新接口拒绝包含 `projectId` 的请求。
- 临时任务页面现有的“归入项目”操作改用专用接口，不受本限制影响。

## 数据模型

### Personnel

- `id`: 唯一标识。
- `name`: 姓名。
- `certificateNumber`: 证书编号，唯一。
- `deletedAt`: 软删除时间，为空表示有效人员。
- `createdAt`: 创建时间。
- `updatedAt`: 更新时间。

### TaskPersonnel

- `taskId`: 任务标识。
- `personnelId`: 人员标识。
- `createdAt`: 分配时间。

`taskId + personnelId` 建立联合唯一约束。任务或人员记录不因人员软删除而级联删除该关联。

### Task 扩展

- `startTime`: 可为空的开始时间。
- 查询任务时附带已分配人员列表，人员项包含删除状态。

## 接口设计

### 人员接口

- `GET /api/personnel`: 返回全部人员，包含已删除人员。
- `POST /api/personnel`: 新增人员。
- `PATCH /api/personnel/[id]`: 编辑未删除人员。
- `DELETE /api/personnel/[id]`: 软删除人员。

接口需要校验必填字段、证书编号唯一性和已删除人员不可编辑规则，并返回明确的中文错误信息。

### 任务更新接口

`PATCH /api/tasks/[id]` 增加：

- `startTime`: ISO 时间字符串或 `null`。
- `personnelIds`: 当前任务完整的人员 ID 列表。

接口校验新分配的人员必须存在且未删除。已关联的已删除人员可以保留或移除，但不能通过请求新增到其他任务。

状态更新为 `in_progress` 且原开始时间为空时，由服务端自动写入当前时间，确保不同页面触发状态流转时行为一致。

临时任务归入项目使用独立的 `PATCH /api/tasks/[id]/assign-project` 接口，该接口只接受目标项目 ID，并校验任务和项目是否存在。常规任务更新接口不再承担项目迁移职责。

## 页面交互

### 设置页面

“测评人员管理”采用列表管理形式：

- 顶部提供姓名、证书编号输入框和添加按钮。
- 列表展示姓名、证书编号、状态、编辑和删除操作。
- 编辑在当前列表区域完成，不跳转新页面。
- 删除前进行普通确认，说明人员将停止出现在新任务分配列表中，但历史任务仍会保留。
- 已删除行弱化显示，并展示“已删除”状态。

### 任务详情面板

- 归属项目显示为只读文本，不再使用可编辑下拉框。
- 增加开始时间输入控件，使用本地日期时间格式编辑。
- 增加人员多选下拉控件，已选人员以可移除标签展示。
- 已删除人员标签显示感叹号标识，并与正常人员有视觉区分。
- 现有任务保存按钮统一提交任务字段、开始时间和人员分配。

## 数据迁移与兼容

- 数据库初始化时创建人员表和任务人员关联表。
- 为任务表增加可空的开始时间列。
- 现有任务默认没有开始时间和分配人员，不影响当前数据。
- 数据导入、导出及备份需要包含人员、任务人员关联和开始时间。

## 测试范围

### 数据层测试

- 新增、编辑和软删除人员。
- 证书编号唯一性校验。
- 已删除人员不可编辑。
- 任务支持分配多名人员。
- 已删除人员仍保留原任务关联。
- 已删除人员不能被新增分配。
- 任务首次进入 `in_progress` 自动记录开始时间。
- 已有或手动修改的开始时间不被状态更新覆盖。
- 常规任务更新不能改变归属项目。

### 接口测试

- 人员 CRUD 的成功与校验失败场景。
- 任务人员列表完整替换、保留及移除已删除人员。
- 非法人员 ID 和已删除人员新增分配的错误响应。

### 页面验收

- 设置页可以新增、编辑和删除人员。
- 删除人员后，新任务下拉列表不再显示该人员。
- 历史任务仍显示 `姓名！`。
- 任务详情可多选人员并保存。
- 首次切换到进行中后自动出现开始时间，且可以手动修改。
- 任务详情不提供项目修改入口。

## 非目标

- 不增加登录、人员账号或权限体系。
- 不记录人员工时、排班或工作负载。
- 不自动重新分配已删除人员的任务。
- 不在本阶段扩展任务模板的默认人员配置。
