# 项目集功能设计

## 目标

在项目管理功能中增加“项目集”，用于集中管理多个相关项目。一个项目最多归属于一个项目集，也可以不归属于任何项目集。项目集不改变现有项目任务、归档和工作流能力，只在项目之上增加一层组织与汇总视图。

## 范围

首期包含以下能力：

- 新建、重命名、删除项目集。
- 查看项目集的项目数量、状态、整体进度和时间范围。
- 查看项目集内的项目列表。
- 将现有项目加入项目集、移出项目集或移动到另一个项目集。
- 在项目集内新建项目，并自动建立归属关系。
- 在项目详情中设置项目开始时间和所属项目集。
- 保留固定的“未分组项目”入口，用于展示未加入项目集的项目。

首期不包含项目集描述、项目集手动状态、项目集手动日期、项目集层级嵌套或一个项目加入多个项目集。

## 数据模型

新增 `ProjectSet`：

- `id`：唯一标识。
- `name`：项目集名称，必填，允许重名。
- `sortOrder`：展示顺序。
- `createdAt`、`updatedAt`：创建与更新时间。

扩展 `Project`：

- `projectSetId`：可空外键。为空表示未分组；一个项目只能关联一个项目集。
- `startDate`：项目开始日期，可空。

项目集的状态、进度、开始时间和结束时间均在查询时动态汇总，不在数据库中重复存储。删除项目集时，其项目的 `projectSetId` 统一置空，项目本身及项目下的任务不被删除或归档。

运行时数据层继续使用 `src/lib/store.ts` 中的 SQLite 实现，同时同步更新 `prisma/schema.prisma` 作为模型参考。数据库初始化需要兼容已有本地数据库，通过增量建表和补列完成升级，不破坏现有数据。

## 自动统计规则

项目集只统计其所属的未归档项目，归档项目保留归属关系但不参与汇总；恢复后自动重新参与。

- 项目数量：未归档项目数量。
- 整体进度：未归档项目进度的算术平均值，四舍五入为整数。
- 开始时间：所有有效项目开始时间中的最早日期。
- 结束时间：所有有效项目截止日期中的最晚日期。
- 缺少开始时间或截止日期的项目仍参与数量、状态和进度统计，但不参与对应日期边界计算。
- 空项目集：状态为“未开始”，进度为 `0%`，起止时间显示“待确定”。
- 完成：项目集内存在未归档项目，且这些项目全部完成。
- 进行中：未达到完成条件，且至少一个项目为进行中。
- 暂停：没有进行中项目，且至少一个项目为暂停。
- 未开始：项目集为空，或项目集内没有可归入上述状态的有效项目。

“未分组项目”是固定的虚拟分组，不写入 `ProjectSet` 表。它显示所有 `projectSetId` 为空且未归档的项目，不需要汇总为一个可编辑的项目集实体。

## 页面结构

项目管理采用三级浏览结构：

### 1. 项目集列表

进入项目管理后，首先展示项目集卡片和固定的“未分组项目”卡片。

项目集卡片显示：

- 项目集名称。
- 未归档项目数量。
- 自动计算的状态。
- 自动计算的整体进度。
- 自动计算的起止日期。

页面支持新建项目集。项目集卡片提供重命名和删除入口；删除前必须明确提示“项目集将被删除，所属项目会移入未分组项目，项目及任务不会被删除”。

### 2. 项目集详情

点击项目集后，仅展示该项目集中的项目，隐藏其他项目集。顶部提供返回项目集列表、重命名和删除项目集的操作。

项目集详情支持：

- 新建项目，新项目自动加入当前项目集。
- 从现有项目中多选加入。
- 将项目移出到未分组项目。
- 将已属于其他项目集的项目移动到当前项目集。

点击“未分组项目”进入同样的项目列表视图，但不显示重命名和删除操作。在其中新建项目时，项目保持未分组。

### 3. 项目详情

项目详情和任务管理区域保持现有结构，增加：

- “项目开始时间”日期字段。
- “所属项目集”下拉框，可选择某个项目集或“未分组项目”。

项目详情的返回按钮返回进入该详情前所在的项目集项目列表。如果项目归属在详情中被修改，保存后返回按钮指向新的所属项目集；改为未分组时返回未分组项目列表。

## 交互与状态管理

前端在现有 `selectedProjectId` 之外增加当前项目集选择状态。项目管理视图按以下状态渲染：

- 未选择项目集、未选择项目：项目集列表。
- 已选择项目集、未选择项目：该项目集的项目列表。
- 已选择项目：项目详情与任务区域。

项目集和项目写操作成功后统一刷新 dashboard 数据，使项目集统计、项目列表及项目详情保持一致。移动项目仅更新 `projectSetId`，不得修改项目任务、状态、进度或排序。

## 接口设计

新增：

- `GET /api/project-sets`：返回项目集及动态汇总结果。
- `POST /api/project-sets`：新建项目集。
- `PATCH /api/project-sets/[id]`：重命名项目集。
- `DELETE /api/project-sets/[id]`：删除项目集，并将所属项目移入未分组项目。

扩展项目接口：

- `POST /api/projects` 接受 `projectSetId` 和 `startDate`。
- `PATCH /api/projects/[id]` 接受 `projectSetId` 和 `startDate`。
- dashboard 项目数据返回 `projectSetId` 和 `startDate`，并增加项目集汇总数据。

校验规则：

- 项目集名称去除首尾空格后不能为空。
- `projectSetId` 非空时必须对应现存项目集，否则返回客户端错误。
- 删除不存在的项目集返回未找到错误。
- 删除项目集和解除项目归属在同一数据库事务中完成。

## 错误处理

- 创建、重命名、删除或移动失败时，保留当前页面和用户输入，并展示明确错误信息。
- 删除项目集必须经过浏览器确认，不允许静默删除。
- 并发情况下如果目标项目集已被删除，项目移动请求失败，项目保持原归属。
- 项目集统计不依赖缓存字段，避免项目状态、日期或归档变化后出现过期汇总。

## 测试方案

### 领域与数据层

- 空项目集的状态、进度和日期。
- 多项目平均进度的四舍五入。
- 最早开始时间和最晚结束时间。
- 完成、进行中、暂停和未开始状态计算。
- 缺少日期的项目不影响另一侧日期边界。
- 归档项目被排除，恢复后重新纳入。
- 一个项目只能归属于一个项目集。
- 项目在项目集之间移动及移入未分组。
- 删除项目集后项目保留、任务保留且项目变为未分组。
- 无效项目集标识被拒绝。

### 接口

- 项目集增、改、删、查。
- 新建项目自动加入当前项目集。
- 项目开始时间和项目集归属的创建与更新。
- 删除项目集事务行为。

### UI

- 项目集列表、项目列表、项目详情三级导航。
- “未分组项目”固定入口。
- 项目集卡片汇总信息展示。
- 多选加入现有项目及跨项目集移动。
- 项目详情切换所属项目集。
- 删除项目集的明显提示及删除后的页面回退。
- 桌面与移动视口下无横向溢出或控件遮挡。

## 验收场景

1. 创建“银行测评项目集”。
2. 将两个现有项目加入该项目集，并在其中新建第三个项目。
3. 为三个项目设置开始时间、截止日期和不同状态。
4. 在项目集卡片查看自动汇总的数量、状态、进度和时间范围。
5. 将一个项目移动到另一个项目集，再将其移入未分组项目。
6. 归档一个项目，确认项目集汇总将其排除；恢复后重新纳入。
7. 删除项目集，确认其中项目全部出现在未分组项目中，项目及任务数据保持完整。
